## BIOS
1. Advanced
- `Restore AC Power Loss` тАФ **Power Off**
- `Wake on LAN` тАФ **Enabled**
- ACPI Settings:
    - `Enable ACPI Auto Configuration` тАФ **disabled**. Let the OS manage ACPI rather than BIOS auto-config, which can conflict with Linux.
    - `Enable Hibernation` тАФ **disabled**. Server, no need, and S4 sleep state can cause issues.
    - `ACPI Sleep State` тАФ **suspend disabled**. You don't want a server hibernating.
    - `Lock Legacy Resources` тАФ **disabled**. Locking legacy IRQ/DMA resources can interfere with PCIe device assignment.
- S5 RTC Wake Settings:
    - `Wake system from S5` тАФ **disabled**
- PCI Subsystem Settings:
    - `Above 4G Decoding` тАФ **enabled**
    - `SR-IOV Support` тАФ **enabled**
    - `PCI Latency Timer` тАФ **default** (32 or 64). Not relevant for PCIe devices.
    - `PCI-X Latency Timer` тАФ **default**. Legacy setting.
    - `VGA Palette Snoop` тАФ **disabled**. Legacy VGA thing, irrelevant.
    - `PERR#/SERR# Generation` тАФ **disabled**. Enabling these can cause system halts on non-fatal PCIe errors, which AMD GPUs occasionally generate harmlessly.
    - `BME DMA Mitigation` тАФ **disabled**. This restricts bus-master DMA before IOMMU is fully initialized тАФ sounds good in theory but can interfere with passthrough. With IOMMU properly configured you don't need it.
    - `Relaxed Ordering` тАФ **enabled**. Improves PCIe throughput.
    - `Extended Tag` тАФ **enabled**. Allows more outstanding PCIe transactions, helps GPU performance.
    - `No Snoop` тАФ **enabled**. GPU drivers use this, disabling it can hurt performance.
    - `Maximum Payload` тАФ **Auto** or **256B**. Larger is generally better for throughput; Auto lets the device negotiate.
- Network Stack Configuration:
    - `Network Stack` тАФ **enabled**
- CSM Configuration:
    - `CSM Support` тАФ **disabled**
        - Had to switch `Video` to **uefi** and reboot so I could actually change `CSM Support` to **disabled**
2. IntelRCSetup
- Processor Configuration:
    - `Check CPU BIST result` тАФ **enabled**. Self-test on boot, harmless.
    - `Monitor/Mwait` тАФ **enabled**. OS and hypervisor use this for efficient CPU idle states.
    - `Execute Disable Bit` тАФ **enabled**. Security feature, Proxmox expects it.
    - `Intel TXT` тАФ **disabled**. Trusted Execution Technology adds complexity with no benefit for your use case, can interfere with virtualization.
    - `VMX` тАФ **enabled**. This is VT-x. This is what you were looking for.
    - `Enable SMX` тАФ **disabled** unless you're using TXT. Paired with TXT, not needed.
    - `Lock Chipset` тАФ **enabled** (default). Security hardening, no impact.
    - `MSR Lock Control` тАФ **enabled**.
    - `PPIN Control` тАФ **default**. Processor serial number access, irrelevant.
    - `Debug Interface` тАФ **disabled**. Security risk if left open.
    - `Hardware Prefetcher` тАФ **enabled**. Helps inference workload memory access patterns.
    - `DCU Mode` тАФ **default**.
    - `Direct Cache Access (DCA)` тАФ **enabled**. Reduces latency for network/PCIe DMA.
    - `DCA Prefetch Delay` тАФ **default** (typically 32).
    - `x2APIC` тАФ **enabled**. Required for systems with many vCPUs, Proxmox benefits from it.
    - `Targeted SMI` тАФ **default**. SMI routing optimization, no meaningful impact.
- Advanced Power Management Configuration:
    - `Power Technology` тАФ **Custom**. "Energy Efficient" lets the CPU throttle, bad for consistent inference latency.
    - `Config TDP` тАФ **default**.
    - `IOTG Setting` тАФ **default**, irrelevant to your use case.
    - `Uncore CLR Freq OVRD` тАФ **default** unless you're overclocking, which you're not.
    - `CPU P State Control` тАФ **enabled**, then set to `HWP Native Mode` if available. Allows the OS to control P-states directly, better than BIOS-managed.

    - `CPU HWPM State Control` тАФ **enabled**. Works with P-state control above.
        - `Enable CPU Autonomous C-State` тАФ **disabled** (Disable it. Autonomous C-state allows the CPU to make its own C-state decisions independent of OS control тАФ counterproductive when you've just manually configured C-states above. )

    - `CPU C State Control` тАФ go in here, disable C6 and C7. Leave C1/C1E enabled.
        - `C2C3TT` тАФ **default**
        - `Package C State Limit` тАФ `C2` to prevent deep package sleep states
        - `CPU C3 Report` тАФ **disabled**
        - `CPU C6 Report` тАФ **disabled**
            - `CPU C3 Report` and `CPU C6 Report` control whether the CPU reports those C-states as available to the OS. If C6 was enabled, the OS can put cores into C6 deep sleep тАФ disable it for consistent latency on a server.

    - `CPU T State Control` тАФ **disabled** (disable T-states entirely. T-states are throttling via duty cycles, never useful for a server).
        - `ACPI T-States` тАФ **disabled**. T-states are CPU throttling via clock modulation тАФ never useful for a server.
    - CPU Advanced PM Tuning:
        - Energy Perf BIAS:
            - `Energy Performance Tuning` тАФ **disabled**. You want manual control, not auto-tuning.
            - `Energy Performance Bias Setting` тАФ will unlock when tuning is disabled; set to `Performance`
            - `Power/Performance Switch` тАФ **disabled** once bias is set to Performance
            - `Workload Configuration` тАФ **I/O Sensitive** (best for inference with GPU DMA traffic)
            - `Average Time Window` тАФ **default** (23)
            - `P0 TotalTimeThreshold Low` тАФ **default** (35)
            - `P0 TotalTimeThreshold High` тАФ **default** (58)
    - `SOCKET RAPL Config` тАФ **default** (RAPL is power capping. Leave defaults. Don't set aggressive power limits or inference will throttle.)
    - `DRAM RAPL` тАФ **default**, leave defaults.
- QPI Configuration:
    - QPI General Configuration:
        - `Link Freqency Select` тАФ **9.6GB/s** (Check that QPI Link Speed is set to maximum (Fast or 9.6 GT/s for E5-2680v4). QPI is the CPU-to-chipset bus, you want it at full speed.)
- Memory Configuration:
    - Memory Thermal:
        - `Memory Power Saving Mode` тАФ **disabled** (disable any memory power saving options. DRAM power saving introduces latency spikes, same reasoning as CPU C-states.)
- IIO Configuration:
    - `PCIe Train by BIOS` тАФ **enabled** (Yes, leave it)
    - `PCIe Hot Plug` тАФ **disabled**, correct
    - `PCIe ACPI Hot Plug` тАФ **disabled**, correct
    - `EV DFX Features` тАФ **disabled**, correct (debug feature)
    - `IOAT Configuration` тАФ **enabled** (go in here, enable IOAT (I/O Acceleration Technology). Helps with DMA throughput for your workload)
        - `Enable IOAT` тАФ **enabled**
        - Just enable IOAT and leave everything else in there at default.

    - `Intel VT for Directed I/O (VT-d)` тАФ **enabled**. This is what you were looking for earlier
    - `PCI Express Global Options`:
    - `TX EQ WA` тАФ **enabled** (PCIe signal equalization workaround)
    - `DMI Vc1/Vcp/Vem Control` тАФ **disabled**
    - `VCO No-Snoop Configuration` тАФ **disabled**
    - `Gen3 Phase3 Loop Count` тАФ **default** (16)
    - `Skip Halt on DMI Degradation` тАФ **disabled**
    - `Power Down Unused Ports` тАФ **enabled** (Yes, leave it)
    - `SLD WA Revision / Rx Clock WA` тАФ **default**
    - `PCI-E ASPM (Global)` тАФ **disabled**. This is the global ASPM toggle, critical for passthrough stability
- PCH Configuration:
    - PCH Devices:
        - `DeepSx Power Policies` тАФ **disabled**, correct
        - `GPe Wake From DeepSx` тАФ **disabled**, correct
        - `SMBUS Device` тАФ **enabled**, leave it
        - `PCH Server Error Reporting` тАФ **disabled**, leave it
        - `PCH Display` тАФ **enabled**, leave it
        - `Serial IRQ Mode` тАФ `Quiet` is fine
        - `External SSC Enable` тАФ **disabled**, leave it
        - `PCH State After G3` тАФ `S0` means it powers on after power loss. Good for a server, leave it
        - `PCH CRID` тАФ **disabled**, leave it
    - PCI Express Configuration
        - `PCI-E ASPM SUPPORT (GLOBAL)` тАФ **disabled** (ASPM: Disable it. ASPM (Active State Power Management) on PCIe can cause passthrough instability and intermittent GPU resets. Not worth the minimal power saving.)
        - Azalea is an AMD BIOS feature for memory training optimization on EPYC/Threadripper platforms. Not relevant to your X99/E5-2680v4 setup тАФ you won't see it, and if you do, ignore it.
    - Platform Thermal Configuration
        - `PCH Thermal Device` тАФ **enabled** (just helps with thermal reporting)
        - `alert enable lock` тАФ **disabled** (locks the the thermal alert threshold so it can't be changed at runtime by software, no reason to use this)
    - USB Configuration
        - `USB Precondition` тАФ **default** (disabled). Speeds up USB init but can cause enumeration issues.
        - `xHCI mode` тАФ **Smart Auto** or **Auto** is fine. Don't set to disabled.
        - `BTCG` тАФ **enabled**, it's a PCH clock gating power optimization, harmless.
        - `Per-port disable control` тАФ **disabled** unless you need to physically disable specific ports.
        - `xHCI idle L1` тАФ **default**. Minor power saving, no real impact.
        - `S755 WA / Interrupt Remap WA` тАФ **Auto** or **enabled**. These are workarounds for specific USB errata; they exist for a reason.
